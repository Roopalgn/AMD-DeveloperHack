# AMD Developer Hackathon

- **Date/Time (listed):** Monday, May 4, 2026, 7:30 PM IST
- **Theme:** Build AI agents and high-performance AI applications on AMD GPUs in the cloud
- **Main Offer:** Start with **$100 AMD Developer Cloud credits** (for eligible AMD AI Developer Program members)

## Overview

The AMD Developer Hackathon is a hands-on event for developers, founders, engineers, and builders exploring real AI systems on AMD infrastructure.

- Build with **AMD Developer Cloud**, **ROCm**, and model APIs
- Focus on real, end-to-end applications and practical AI workflows
- Open to solo participants and teams

## Eligibility and Participation

- To unlock credits and join, sign up for the **AMD AI Developer Program**
- Participation is online on lablab.ai + Discord, with an invited on-site phase in San Francisco
- On-site participation is by invitation only
- Travel and accommodation are not covered

Build an application, AI agent, or developer tool that is realistic, end-to-end, and demonstrates AMD compute capabilities.

## AMD Developer Cloud Getting Started (Consolidated)

This section summarizes the included guide "How to Get Started on the AMD Developer Cloud" (Jun 30, 2025).

### VM Setup Flow
1. Choose GPU plan (single MI300X or 8x MI300X node; usage billed in GPU-hours)
2. Choose image:
   - Bare OS (manual setup)
   - Quick Start ROCm-ready images (optional preloaded AI stacks)
3. Add SSH key and create VM
4. Access VM via:
   - SSH (`ssh root@<public-ip>`)
   - Web Console
   - JupyterLab (token-based)

### Quick Start Package Notes
- Example package images may include vLLM, SGLang, PyTorch, Megatron, and JAX
- Some images launch workloads inside Docker; common command:
  - `docker exec -it rocm bash`

### Jupyter and Notebook Workflow
- Quick Start images can include ready-to-use notebooks
- Typical flow:
  1. Open notebook
  2. Start vLLM server from terminal tab
  3. Run notebook cells step by step

### Complimentary Credit Notes
- Initial offer referenced: 25 complimentary GPU-hours for approved users
- Must be used within the stated validity window after approval
- If credits are exhausted, GPU/data access may be affected
- Powered-off VMs may still incur charges for reserved resources
- Destroy instances when finished; snapshots can be used for pause/resume
